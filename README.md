# Multicompartment inspiratory energy-routing model

Computational code supporting the manuscript:

> **Inspiratory Temporal Patterning and Mechanical Architecture Shape Regional Energy Distribution in a Multicompartment Lung Model**

This repository implements a deterministic, linear, six-compartment parallel resistance–compliance model used to examine how inspiratory waveform, inspiratory time, end-inspiratory pause, and the compartment-level arrangement of resistance and compliance affect regional inspiratory energy allocation.

## Scope

The six compartments are abstract mechanical pathways exposed to a common airway-opening pressure; they are not intended to represent specific anatomical lung regions.

Three prespecified aggregate-matched mechanical architectures are evaluated:

- **compliance-dominant:** increasing compliance with fixed resistance;
- **resistance-dominant:** increasing resistance with fixed compliance;
- **mixed:** the same resistance and compliance gradients arranged using the prespecified inverse pairing.

All three architectures have a total compliance of 0.040 L/cmH₂O and a parallel-equivalent resistance of 8 cmH₂O·s/L.

The main factorial analysis contains 54 scenarios:

- 3 mechanical architectures;
- 3 inspiratory waveforms: square, decelerating, and sinusoidal;
- 3 inspiratory times: 0.6, 1.0, and 1.5 s;
- 2 end-inspiratory pause fractions: 0 and 0.2.

An additional exhaustive sensitivity analysis evaluates all 6! = 720 possible pairings of the six resistance values with the ordered compliance profile.

## Model equations

For compartment *i*, flow is defined as:

```text
q_i(t) = [P_aw(t) - PEEP - V_i(t)/C_i] / R_i
```

with conservation of imposed total inspiratory flow:

```text
Q(t) = sum_i q_i(t)
```

Airway-opening pressure above PEEP is therefore:

```text
P_above(t) =
[Q(t) + sum_i V_i(t)/(C_i R_i)] / sum_i 1/R_i
```

Compartment volume is advanced using explicit forward Euler integration:

```text
V_i(t + dt) = V_i(t) + q_i(t) dt
```

Global and compartmental inspiratory energies are accumulated from pressure above PEEP and flow. Mechanical power is calculated as global inspiratory energy per breath multiplied by respiratory rate.

## Reported metrics

- **REF** — regional energy fraction: compartmental inspiratory energy divided by global inspiratory energy.
- **EII** — energy inequality index: population coefficient of variation of the six REF values.
- **DCES** — dominant compartment energy share: maximum REF.
- **MP** — global inspiratory mechanical power in J/min.

These quantities are model-based energy-routing descriptors. They are not direct measurements of tissue stress, strain, injury, or clinical risk.

## Requirements

Python 3.10 or newer is recommended.

Create and activate a virtual environment, then install the dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
python3 -m pip install -r requirements.txt
```

On Windows PowerShell, activate the environment with:

```powershell
.venv\Scripts\Activate.ps1
```

## Reproducing the analyses

Run the scripts in the following order from the repository root:

```bash
python3 analysis_pipeline.py
python3 pairing_sensitivity_analysis.py
python3 validate_outputs.py
```

There is intentionally no wrapper script such as `run_all.py`; each computational stage can be run and inspected independently.

## Validation

After regenerating both groups of outputs, run:

```bash
python3 validate_outputs.py
```

The validator checks:

- aggregate matching of the three mechanical architectures;
- completeness and uniqueness of the 54-scenario factorial grid;
- REF normalization;
- tidal-volume, flow, and energy conservation;
- reference-condition results recalculated directly from the model;
- completeness and uniqueness of all 720 resistance–compliance pairings;
- consistency of stored time constants with `tau_i = R_i C_i`;
- the reported positive and negative inspiratory-time response counts.

Optional derived tables are also checked when present.

A plain-text validation report is written to:

```text
outputs/VALIDATION_REPORT.txt
```

## Numerical implementation

The default simulation time step is 0.001 s.

Prescribed inspiratory flow is normalized using a left-rectangle sum. Pressure and energy are evaluated at interval left endpoints, and compartment volumes are advanced using explicit forward Euler integration.

The analysis pipeline repeats the complete main factorial grid using a time step of 0.0005 s for numerical-convergence assessment.

Each simulation starts from zero compartment volume above end-expiratory volume and covers inspiration, including any zero-total-flow end-inspiratory pause.

Expiration and cumulative multibreath dynamics are not simulated.

## Reproducibility notes

- The workflow is deterministic and does not use random sampling.
- Running the analysis scripts overwrites locally generated output files.
- The exhaustive pairing analysis requires `numba` and `scipy` in addition to `numpy` and `pandas`.
- Numerical outputs should remain reproducible within floating-point precision.
